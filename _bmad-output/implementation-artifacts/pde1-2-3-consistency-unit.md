# Story 2.3 (PRD 1, Epic 2): Consistency unit and fingerprint

Status: done

## Story

As a self-service provisioner, I want a provable "same environment" identity across teams, so
that I can trust two datasets are the same synthetic reality (PDE-11, AD-15). Closes Epic 2.

## Acceptance Criteria

1. Given a Spec bundling pinned Profile artifacts, when I regenerate, then generation reuses the
   bundled Profiles offline and never re-profiles the source (AD-15).
2. The run emits a consistency-unit fingerprint = a hash over (Spec + pinned Profiles + seed +
   pinned deps); identical units produce byte-identical output and identical fingerprints
   (PDE-11).

## Tasks

- [ ] `tymi/config/consistency.py` — `pinned_deps()` (generation-affecting package versions),
  `consistency_fingerprint(spec, *, deps=None)` (stable hash over the Spec's canonical dump +
  pinned deps).
- [ ] Unit tests: same unit → same fingerprint + byte-identical `generate_from_spec` output;
  seed / shared-key / reserved-block / profile / deps change → different fingerprint; offline
  reuse (YAML round-trip, no source) keeps the fingerprint stable and still generates.
- [ ] Full 3-layer `bmad-code-review` gate.

## Dev Notes

- **The unit is already mostly in the Spec.** `Spec.model_dump(mode="json")` bundles the pinned
  Profiles (each with its fixed salt), seed, tolerance, shared-key decls, reserved blocks, and
  fixtures — i.e. everything in AD-15 except pinned deps. The fingerprint hashes
  `{spec_dump, deps}` canonically (sorted keys, blake2b).
- **Offline reuse (AC1) is already the behavior** — `generate_from_spec` → `spec_profiles`
  reconstructs Profiles from the bundled dicts and takes no adapter, so it never re-profiles. The
  test pins this by round-tripping through YAML with no source and asserting a stable fingerprint.
- **Why deps matter:** numpy/pandas/faker/tymi versions can shift generated bytes, so a changed
  dep must change the fingerprint (signal potential divergence) — the consistency unit includes
  them. Emission into the provisioning *report* is Story 3.3 (PDE-15); this story supplies the
  fingerprint function it will call.
