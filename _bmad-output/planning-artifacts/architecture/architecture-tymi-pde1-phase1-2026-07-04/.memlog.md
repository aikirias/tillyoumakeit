- (decision) AD-14: the Spec is a versioned whole-DB superset of Config = {per-table Profiles set + sensitive marks + fixtures + shared-key declarations + seed + tolerances}, one artifact with schema_version. Binds PDE-2/3, extends AD-5. Prevents divergent spec formats / where fixtures+sensitive live. Rule: provision takes exactly one Spec; the Spec embeds pinned Profile snapshots (AD-15).
- (decision) AD-15: identity = the CONSISTENCY UNIT (Spec + pinned source-Profile snapshots + seed + pinned deps), not spec+seed. Binds PDE-11/G4. Prevents 'same spec, drifted/different source -> mismatched entities -> garbage joins'. Rule: the Spec pins the Profile snapshots it was bootstrapped from; regeneration reuses them OFFLINE (never re-introspects a drifted source); the provisioning report emits a consistency-unit fingerprint (hash of the tuple).
- (decision) AD-16 [ASSUMPTION - confirm OQ-5]: designated SHARED entity keys are position-deterministic + source-independent, in a keyspace partitioned away from fixtures. Binds PDE-12/10. Prevents divergent key gen / cross-team drift / fixture collision. Rule: columns declared 'shared' in the Spec are emitted as a deterministic monotonic sequence by (table,row-position) [ratifies existing _unique_column arange], NOT sampled from source; the Spec pins per-table row counts so keys align across teams; a reserved keyspace block is set aside for fixtures.
- (decision) AD-17: fixtures = inject-verbatim (post-generation overlay in the reserved fixture keyspace), regenerate-NEVER, check-ALWAYS. Binds PDE-8/9/10. Prevents fixtures-as-PII-bypass. Rule: fixtures skip obfuscation/regeneration but ARE passed through the leakage gate + PII scan (a real PII value in a fixture fails closed); adding a fixture requires a logged attestation; synthetic fixtures preferred.
- (decision) AD-18 [ASSUMPTION - confirm OQ-2]: provision fails closed on destination unless explicitly affirmed non-prod; obfuscation never optional. Binds PDE-14/NFR-E/CM2. Prevents writing to prod / a soft obfuscate checkbox. Rule: destination config must carry an explicit environment=nonprod affirmation AND not match a configurable prod deny-list (host/db patterns); missing affirmation or a deny-list match aborts BEFORE any write; no code path loads un-obfuscated source into a destination.
- (decision) AD-19: provision is a single thin in-process pipeline invoked by an external scheduler; no scheduling/retry/DAG logic in the library. Binds PDE-13, extends AD-8. Prevents orchestration creep into core; CLI and DAG re-implementing the flow. Rule: 'tymi provision --spec' runs load-Spec -> pinned Profiles -> generate_related -> leakage gate -> fixtures overlay -> load, as one core pipeline function the CLI and any DAG call identically; retries/scheduling live in the external scheduler.
