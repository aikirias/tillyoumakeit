- (constraint) PRD: Python 3.11+; CLI + library + web UI with parity via one YAML declarative config.
- (constraint) PRD: 4 DB engines equal priority, source<->destination interchangeable (MSSQL, StarRocks, PostgreSQL, MySQL) behind SourceConnector.
- (constraint) PRD NFR-1: zero leakage, enforced by exact membership check (not sampling); Profiles store no re-identifiable raw values.
- (constraint) PRD NFR-4: seed reproducibility byte-for-byte on deterministic exporters.
- (constraint) PRD NFR-5: new Source engine or new Mutator added via plugin without touching core.
- (constraint) PRD NFR-2: profiling by sampling/streaming, tables >=10M rows without exhausting memory.
- (direction) User is non-expert: follow best practices, prefer a paved-path single-language (Python) stack, minimize moving parts. Deliverable: spine + solution design + simple summary.
- (decision) AD-1 Hexagonal (Ports & Adapters): domain core depends only on Port interfaces; all I/O (DB/files/CLI/UI) in adapters; deps point inward. Prevents core coupling to any engine/UI.
- (decision) AD-2 EngineAdapter is bidirectional (introspect+sample AND load); source and destination are two runtime roles of one interface. Prevents separate source/sink hierarchies breaking any->any.
- (decision) AD-3 Extensibility via Python entry points (groups tymi.engines, tymi.mutators); core never imports a concrete adapter/mutator. Serves NFR-5.
- (decision) AD-4 Single numpy Generator seeded from config.seed, threaded explicitly to every stochastic component; no global random/time seeds. Serves NFR-4.
- (decision) AD-5 Declarative Config (Pydantic v2 + YAML, semver schema_version) is single source of truth; CLI and UI build the same Config and call the same orchestrator; loader rejects unknown major.
- (decision) AD-6 Profile stores only aggregates (histograms/quantiles/freqs/correlations/patterns)+schema+PII tags, never raw values; free-text only pattern/length stats. Serves NFR-1.
- (decision) AD-7 Zero-leakage via exact membership check: sensitive columns checked against a hashed set of real source values, regenerate on collision; final exact gate. Serves NFR-1/SM-2.
- (decision) AD-8 Pipes-and-filters pipeline orchestrated in core: Connect->Profile->Generate(Faithful|Chaos)->PrivacyFilter->Evaluate->Export; CLI/UI only build Config + invoke.
- (decision) AD-9 Permissive-license-only dependency policy (MIT/Apache/BSD/LGPL-dynamic). SDV (BUSL-1.1) explicitly excluded; every new dep license-checked.
